# CoincallTrader — Project Context for AI Agents

**Version:** 1.6.0
**Last Updated:** 28 March 2026
**Python:** 3.9+ (no 3.10+ syntax — use `Optional[X]`, not `X | None`)
**Venv:** `.venv` (Python 3.9.6) — activate before any terminal command

Automated BTC/ETH options trading bot with multi-exchange support (Coincall + Deribit).
Config-driven strategies, tick-based execution, slot-based multi-strategy deployment.

---

## 1  Architecture

```
PositionMonitor (10s poll, daemon thread)
  │
  ├─► LifecycleEngine.tick(snapshot)   — advances every active trade's state machine
  │
  └─► StrategyRunner.tick(snapshot)     — per strategy:
        1. check_closed_trades()        — fire on_trade_closed callback
        2. entry gate (all conditions)  — time, weekday, margin, equity, delta
        3. resolve_legs()               — LegSpec → concrete TradeLeg via option_selection
        4. lifecycle_engine.create()    — creates TradeLifecycle with exit conditions
        5. lifecycle_engine.open()      — routes to limit / rfq executor
```

### Trade State Machine

```
PENDING_OPEN → OPENING → OPEN → PENDING_CLOSE → CLOSING → CLOSED
                                                           └─► FAILED
```

Exit conditions are evaluated every tick while OPEN. Any single exit returning True triggers PENDING_CLOSE.

### Design Principles
1. Composition over inheritance — callables, not class hierarchies
2. Dataclasses everywhere — typed data containers, frozen for thread-safe snapshots
3. Single responsibility — one module per concern
4. Tick-driven — no event queues or extra threads (except PositionMonitor + dashboard)
5. Side encoding: all internal code uses `"buy"` / `"sell"` strings. Exchange adapters convert at API boundary.

## 2  Exchange Abstraction

### ABC Interfaces (`exchanges/base.py`)
- `ExchangeAuth` — `reachable`, `get()`, `post()`, `is_successful()`
- `ExchangeMarketData` — `get_index_price()`, `get_option_instruments()`, `get_option_details()`, `get_option_orderbook()`
- `ExchangeExecutor` — `place_order(symbol, qty, side: str, ...)`, `cancel_order()`, `get_order_status()`
- `ExchangeAccountManager` — `get_account_info()`, `get_positions()`, `get_open_orders()`
- `ExchangeRFQExecutor` — `execute()`, `execute_phased()`, `get_orderbook_cost()`

### Factory (`exchanges/__init__.py`)
`build_exchange(name)` → returns dict with `auth`, `market_data`, `executor`, `account_manager`, `rfq_executor`, `state_map`. Reads `config.EXCHANGE` (default `"coincall"`).

### Coincall Adapter (`exchanges/coincall/`)
Thin delegation wrappers — no business logic. `CoincallExecutorAdapter` converts `"buy"`/`"sell"` → `1`/`2` ints. Auth: HMAC-SHA256.

### Deribit Adapter (`exchanges/deribit/`)
Full implementation — all 7 modules complete:
- **Auth:** OAuth2 client_credentials, proactive token refresh at 80% TTL, thread-safe
- **JSON-RPC:** All calls via `DeribitAuth.call(method, params)` — JSON-RPC 2.0 envelope, Bearer token for private methods
- **Price normalization:** BTC-denominated → USD conversion at adapter boundary
- **Tick sizes:** `_snap_to_tick()` — price < 0.005 BTC → tick 0.0001; ≥ 0.005 → tick 0.0005
- **Symbol translation:** `coincall_to_deribit()` / `deribit_to_coincall()` — e.g. `BTCUSD-03APR26-74000-C` ↔ `BTC-3APR26-74000-C`
- **RFQ:** Block RFQ create→poll→accept; 25 BTC minimum
- **Env vars:** `DERIBIT_CLIENT_ID`, `DERIBIT_CLIENT_SECRET`, `DERIBIT_BASE_URL`
- **URLs:** testnet `https://test.deribit.com`, prod `https://www.deribit.com`

## 4  Strategies

All strategies use `_p("NAME", default, cast)` helper → reads `PARAM_<NAME>` env vars.
Register in `strategies/__init__.py`, select via `SLOT_STRATEGY` env var.

| File | What it does | Exchange | Status |
|------|-------------|----------|--------|
| `strategies/daily_put_sell.py` | Sell 1DTE OTM put at -0.10 delta, 03-04 UTC. Fair-price SL (70%), phased execution (RFQ → limit fair → limit bid+33% → limit bid). EMA-20 filter, liquidity guard. | Coincall | **Active** (slot-01) |
| `strategies/long_strangle_index_move.py` | Long ±$2000 OTM strangle, exit on $1500 BTC index move or 19:00 UTC. Phased close with 0.0001 BTC floor. | Deribit | **Active** (slot-02) |
| `strategies/atm_straddle_index_move.py` | Long ATM straddle, exit on BTC index move ≥$1200 or time exit | Either | **Complete** |
| `strategies/blueprint_strangle.py` | Template strategy — starting point for new strategies | Either | **Template** |

### Writing a New Strategy
```python
# strategies/my_strategy.py
from strategy import StrategyConfig, time_window, min_available_margin_pct
from option_selection import strangle
from trade_lifecycle import profit_target, max_hold_hours

def my_strategy() -> StrategyConfig:
    return StrategyConfig(
        name="my_strategy",
        legs=strangle(qty=0.01, call_delta=0.25, put_delta=-0.25, dte=1, side="sell"),
        entry_conditions=[time_window(8, 20), min_available_margin_pct(50)],
        exit_conditions=[profit_target(50), max_hold_hours(24)],
        max_concurrent_trades=1,
    )
```

## 5  Execution Pipeline

| Mode | Module | When | How |
|------|--------|------|-----|
| `limit` | `trade_execution.py` | Default / small trades | Per-leg limit orders via `LimitFillManager`. Phased pricing: `ExecutionPhase(pricing, duration, buffer_pct, reprice_interval)`. Best-effort close: skip bad-priced legs rather than aborting. |
| `rfq` | `rfq.py` | $50k+ notional | Atomic multi-leg RFQ. `RFQParams` configures timeout, min improvement, fallback. |
| `auto` | `execution_router.py` | Default mode | Routes by notional: rfq ≥ $50k, else limit. |

## 6  Dashboard & Hub

### Per-Slot Dashboard (`dashboard.py`)
Flask + htmx, daemon thread. Session auth with `DASHBOARD_PASSWORD`, brute-force lockout, `secrets.compare_digest()`.

**Dashboard modes** (`DASHBOARD_MODE` env var): `full` (default UI), `control` (headless, localhost-only `/control/*` endpoints), `disabled`.

**Control endpoints** (localhost-only, used by hub): `/control/status`, `/control/pause|resume|stop|kill`

**Kill switch** (`/api/killswitch`): Two-phase mark-price close via `PositionCloser`.

### Hub Dashboard (`hub/hub_dashboard.py`)
Separate Flask app at `/opt/ct/hub/`, port 8070. Aggregates all slots.
- Discovers slots by scanning `/opt/ct/slot-*` directories
- Proxies control commands to slot localhost ports (808X)
- Tick recorder health card via `RECORDER_HEALTH_PORT`
- Routes: `/`, `/api/overview`, `/api/recorder`, `/api/slot/<id>/detail|logs|pause|resume|stop|kill`

## 7  Slot Architecture & Deployment

### Slots
Multi-strategy deployment via isolated "slots" at `/opt/ct/slot-XX/` on production VPS.
Each slot = full codebase copy + own `.env`, `logs/`, venv, systemd service (`ct-slot@XX.service`).

### Slot Config System
- `accounts.toml` — named exchange accounts mapping to env var names (no secrets)
- `slots/slot-XX.toml` — per-slot config: strategy name, account reference, param overrides
- `slot_config.py` — reads TOML → generates `.env.slot-XX`
- Strategy params: `_p("QTY", 0.8)` reads `PARAM_QTY` from env

### Deploy Commands
```bash
./deployment/deploy-slot.sh 01           # deploy + restart slot-01
./deployment/deploy-slot.sh 01 --setup   # first-time slot setup
./deployment/deploy-slot.sh 01 --logs    # tail slot journalctl
./deployment/deploy-slot.sh hub          # deploy hub dashboard
./deployment/deploy-slot.sh status       # overview of all slots
```

## 8  Backtester (`backtester2/`)

Real-data options backtester using Deribit historical tick data.

### Data Stack
1. **Raw ticks** — Tardis parquets (600-750 MB/day) or tick recorder output
2. **Snapshots** — `snapshot_builder.py` → 5-min option snapshots + 1-min spot OHLC (zstd parquet)
3. **Replay** — `MarketReplay` loads parquets → `MarketState` with O(1) option lookups + pre-computed spot excursion arrays

### Components
| File | Purpose |
|------|---------|
| `engine.py` | `run_grid()`, `run_single()` — single-pass multi-param evaluation |
| `market_replay.py` | `MarketReplay` iterator, `MarketState`, `OptionQuote`, `SpotBar` |
| `strategy_base.py` | Strategy protocol: `configure()` → `on_market_state()` → `on_end()` → `reset()` |

| `snapshot_builder.py` | Raw tick parquets → 5-min option snapshots |

## 9  Tick Recorder (`backtester2/tickrecorder/`)

Live Deribit tick recorder via WebSocket — produces parquets matching backtester2 schema.

**Burst-mode capture:** WebSocket stays connected idle with heartbeat. 10s before each 5-min boundary, subscribes to all `ticker.<instrument>.100ms` channels + `deribit_price_index.btc_usd`. Snapshot 0.2s after boundary. Immediately unsubscribes. Writes zstd parquet.

**Data pipeline:** `recorder.py` (live WS) → daily parquets → `sync.py` (rsync from VPS) → `merge.py` (concatenate ranges) → `backtester2/snapshots/`

| File | Purpose |
|------|---------|
| `recorder.py` | Main asyncio daemon, burst snapshot cycle |
| `ws_client.py` | Persistent unauthenticated WebSocket to Deribit |
| `snapshotter.py` | 5-min aggregation, daily parquet writer, crash recovery |
| `instruments.py` | REST instrument discovery, change callbacks |
| `health.py` | HTTP `/health` endpoint, Telegram alerts (throttled) |
| `config.py` | `RECORDER_*` env vars |
| `sync.py` | rsync download from VPS |
| `merge.py` | Combine daily parquets into range files |

**Deployment:** systemd `ct-recorder.service`. Env: `RECORDER_HEALTH_PORT` (8090), `RECORDER_DATA_DIR`, `RECORDER_SNAPSHOT_INTERVAL_MIN` (5).

## 12  Resilience

- **Request timeouts:** 30s on every API call
- **@retry:** Exponential backoff (1→2→4s) for ConnectionError/Timeout only
- **Error isolation:** Main loop tolerates 10 consecutive errors, then exits + Telegram alert
- **Trade persistence:** Active trades → `logs/trades_snapshot.json` every tick; completed → `logs/trade_history.jsonl`
- **Trade recovery:** On startup, verifies snapshot against exchange positions, re-attaches exit conditions
- **Kill switch:** `PositionCloser` two-phase close (mark → aggressive ±10%)
- **Process supervision:** systemd on production (auto-restart)

## 13  Coding Conventions

- `Optional[X]`, not `X | None` (Python 3.9 compat)
- Dataclasses everywhere, frozen for thread-safe snapshots
- Factory functions for conditions — return callables, not classes
- Logging via `logging.getLogger(__name__)` in every module
- Fire-and-forget for Telegram — never crashes the bot
- Side as string (`"buy"` / `"sell"`) in all internal code
- `_p("NAME", default)` for strategy params from env vars
